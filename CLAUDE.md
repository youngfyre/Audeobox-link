# Audeobox Link

## Overview

JUCE-based VST3/AU plugin that bridges the AudeoboxDesktop Electron app and any DAW. Reads DAW transport state (BPM, play/pause, position) and relays it to the desktop app via TCP. Shows "now playing" info from the desktop app inside the DAW.

**Not an audio effect.** The plugin reads audio levels for its UI meter but clears the output buffer (passes silence). Categorized as `IS_SYNTH TRUE` — loads on instrument tracks.

## Architecture

```
Source/
├── PluginProcessor.h/.cpp    # Audio processor + TCP bridge thread
├── PluginEditor.h/.cpp       # Custom-painted 680x420 UI
```

- **4 source files total** — no subdirectories, no third-party code beyond JUCE
- **TCP bridge:** Connects to `127.0.0.1:48142` (server in AudeoboxDesktop main process)
- **Protocol:** Newline-delimited JSON — `hello` → `helloAck`, then `transport` updates every 100ms
- **No persistent state** — `getStateInformation()` / `setStateInformation()` are no-ops

## CI/CD Pipeline

**Trigger:** Push `v*` tags to `main`

**Workflow:** `.github/workflows/build-and-release.yml` — two separate jobs (not matrix):

### macOS Job
1. Download JUCE 8.0.4 (cached)
2. CMake configure + build (passes version from package.json via `-DAUDEOBOX_VERSION`)
3. Code-sign VST3 + AU with Developer ID Application cert
4. Build `.pkg` installer via `pkgbuild` + `productbuild` (installs to system plugin dirs)
5. Sign `.pkg` with Developer ID Installer cert
6. Notarize `.pkg` with Apple
7. Upload to GCS + Slack notification

### Windows Job
1. Download JUCE 8.0.4 (cached)
2. CMake configure + build
3. Generate Inno Setup `.iss` script via `scripts/package.sh`
4. Build `.exe` installer via Inno Setup
5. Upload to GCS + Slack notification

**GCS:** `gs://audeobox-cdn/vst/releases/`
- Installers: `.pkg` (mac), `.exe` (win)
- Manifests: `latest-mac.yml`, `latest.yml` (electron-builder-compatible YAML format)

**GitHub Secrets:**
- `CSC_LINK` / `CSC_KEY_PASSWORD` — Developer ID Application .p12 (codesign)
- `CSC_INSTALLER_LINK` / `CSC_INSTALLER_KEY_PASSWORD` — Developer ID Installer .p12 (productsign)
- `APPLE_ID` / `APPLE_APP_SPECIFIC_PASSWORD` / `APPLE_TEAM_ID` — notarization
- `GCP_SA_KEY` — GCS service account
- `SLACK_WEBHOOK` — #dev channel

## CMake Configuration

- `AUDEOBOX_VERSION` — cache variable, overridden by CI from package.json
- `JUCE_DIR` — path to JUCE, defaults to `./JUCE`
- Plugin formats: VST3 (all platforms) + AU (macOS only, via `if(APPLE)`)
- `IS_SYNTH TRUE` — categorized as instrument in DAWs
- `HARDENED_RUNTIME_ENABLED TRUE` — required for macOS notarization
- Debug panel: `#if JUCE_DEBUG` — hidden in Release builds

## Version Management

**Two version sources that must stay in sync:**
1. `package.json` `version` — source of truth, bumped by `npm version`
2. `CMakeLists.txt` `AUDEOBOX_VERSION` — defaults to `0.1.0`, overridden by CI

CI passes the package.json version to CMake: `"-DAUDEOBOX_VERSION=${{ steps.version.outputs.version }}"`

**Quoting is critical** — without quotes, CMake on Windows splits version strings like `0.1.10` into separate path arguments.

## TCP Bridge Protocol

Messages are newline-delimited JSON with `type`, `version: 1`, `sentAt` fields.

| Direction | Type | Purpose |
|-----------|------|---------|
| Plugin → Desktop | `hello` | Handshake with plugin version, DAW name |
| Desktop → Plugin | `helloAck` | Confirms connection, assigns sessionId |
| Plugin → Desktop | `transport` | BPM, play/pause, position (every 100ms) |
| Plugin → Desktop | `heartbeat` | Keep-alive (every 1s) |
| Desktop → Plugin | `nowPlaying` | Track title, artwork path, level |
| Desktop → Plugin | `setSyncEnabled` | Toggle sync on/off |
| Desktop → Plugin | `disconnect` | Clean disconnect |
| Plugin → Desktop | `goodbye` | Acknowledges disconnect |
